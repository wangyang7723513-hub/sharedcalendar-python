# 共享日历项目 - Prompt (提示词) 提炼与复用指南

在本次使用 AI 辅助开发“基于 Python 与飞书多维表格后台的极简共享日历”时，我们总结了初版 Prompt 导致在本地调试阶段“前端静默卡死”和“数据格式报错”等问题的原因。
以下是经过改良、可直接复用及调整的高质量 Prompt 模板，旨在帮助您在未来类似的前后端分离开发中，一次性生成具备强大容错与极好体验的代码。

---

## ❌ 初版 Prompt (反面教材)

> 帮我写一个共享日历应用，要求手机可以访问，界面类似苹果风格，极简。
> 后端不用传统数据库，我要用 Python 调飞书的多维表格做存储。
> 前端用 HTML 和 Tailwind CSS。

**缺陷分析：**
1. **缺失本地调试边界限制**：没有声明前后端分离时势必带来的跨域和端口访问问题。
2. **缺乏防御性编程约定**：没有要求前端所有的业务请求（如点击登录、注册、拉取日程）必须要有明显的 UI 反馈（如报错弹窗）以及强制的异常捕获。
3. **未交待外部 API 的严格性**：未告知 AI 飞书接口存在的特殊数据结构（文本变多层列表、必需的 filter 配置等），导致生成的 Python 脚本过于乐观，一跑就报错。

## ✅ 改良后的高质量 Prompt (可直接复印 / 引用模板)

如果您下次需要开发类似系统，或者需要创建一个新的 H5 + 飞书/云端数据库应用，**请直接套用并修改以下 Prompt 结构**：

***

### 🧱 [角色与大目标]
作为一名资深的全栈工程师和高级 UX 设计师，请帮我开发一个**[填写类型，例：团队共享日历 / 任务看板]**应用。
要求具备极致简约的 Apple 风格（大字体、高对比度、平滑毛玻璃动画、微距点按反馈），并且必须做双端响应式适配（PC宽屏与移动端 H5 竖屏）。

### 🛠 [技术栈约束]
- **前端**：只使用纯 HTML5 + JavaScript (ES6+)，样式完全采用 Tailwind CSS（通过 CDN 引入），无需复杂的 Node.js/Vue/React 构建配置。
- **后端**：使用 Python (建议基于 Flask 框架) 构建轻量级服务端代码，作为中间人。
- **数据库生态**：不采用本地 SQLite/MySQL，全量业务数据基于调用 **「飞书开发平台 OpenAPI - 多维表格 Bitable」** 的记录读写接口来实现增删改查。

### 🚨 [必须遵守的开发约定 (Critical Rules)]

**1. 前端防御性交互约束 (非常重要)：**
所有涉及网络请求（Fetch）的按钮交互（例如点击提交 / 登录 / 注册 / 拉取数据）：
* 必须包裹严格的 `try...catch...finally` 块。
* 发起请求前立刻冻结（禁用）按钮并给以视觉 Feedback（比如文字变成 `提交中...`）。
* 在进入异常报错的 `catch` 流程，或者接口返回失败值时，**必须第一时间通过强提示弹窗 `alert(报错信息)` 通知用户**，同时**瞬间恢复按钮**的可用状态，**绝对禁止出现请求报错但界面静默无响应的“假死挂起”情况！**

**2. 跨域与网络通信保障：**
* 请显式地指明局域网开发环境的地址。Python 服务端代码中必须引入并应用 `CORS`，解决前端 Live Server 与 Python API 间的跨域拦截。
* 如果在浏览器由于缓存问题加载了旧版代码，请提供通过加后缀（如 `?v=timestamp`）避开静态文件缓存的指令注释。

**3. 飞书 Bitable OpenAPI 安全接入法则 (防报错)：**
* **条件搜索 `Search` 约束**：无论搜索条件多么简单（哪怕仅仅比对一个用户名），传给飞书的 `filter` JSON 中**必须包含 `conjunction: "and" (或 "or")` 参数**，严禁遗漏。
* **返回值剥离防护**：从飞书拉取的多维表格记录，其“文本型”字段很大可能是一个复杂的对象素组（例如 `[{"text": "张三"}]`），**禁止在后端直接将未清洗的数据传给前端**。请务必用代码在 Python 里做诸如 `val = arr[0].get("text") if isinstance(arr, list) else ""` 的强校验与提取。
* **业务排重意识**：飞书本身是不防重名注册的。编写新增（Create）接口逻辑前，必须先行调用 Search 接口比对唯一性字段，如果已存在，则报错阻断。

### 📋 [核心功能需求]
*(在这里逐条列出您期望这个系统能干什么，例如)*
1. 用户可以通过输入 用户名 和密码实现登录与注册，包含头像配色功能。
2. 登录后能够看到按当月日期排布的月视图日历，展示对应日期的所有团队排班或日程安排（用不同的色块深浅表示人员安排密度），支持上一月/下一月的快速翻页。
3. 点击指定日期可弹出浮层，展示该日期的详细待办事项列表；可在此进行新事物的添加，并且允许创建者删除自己创建的条目。

***

## 💡 使用建议

这套 Prompt 的价值在于其**“防坑思维”**（即前面的 3 个 Critical Rules）。
当你拿着这个模板让 AI 去写诸如 “**报销审批单系统**”、“**家庭记账本**”、“**公司库存登记**” 时，只要它的技术栈也是基于 HTML+Python+飞书多维表格（或其他诸如 Supabase、Notion 那样弱强限制、靠 API 驱动的无代码数据库），它写出来的代码质量、容错率以及前后端调试顺畅度将提升几十倍。
